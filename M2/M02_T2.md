## Why These Features Matter

Quarto ships with a collection of features that transform a plain notebook into a professional publication.
This tutorial walks through the most impactful ones for business analytics reporting.

## Callout Blocks

Callout blocks visually distinguish important content — notes, warnings, tips, and cautions.

### Syntax

`markdown
::: {.callout-note}
Use this for supplementary information.
:::

::: {.callout-warning}
Use this for caveats or edge cases.
:::

::: {.callout-tip}
Use this for best practices.
:::

::: {.callout-important}
Use this for critical requirements.
:::

::: {.callout-caution}
Use this when an action may have unintended side effects.
:::
`

Each callout can have a custom title:

`markdown
::: {.callout-tip title="Model Selection Advice"}
For small datasets (< 10,000 rows), prefer logistic regression over neural networks.
:::
`

Callouts support collapse: true to hide body by default:

`markdown
::: {.callout-note collapse=true}
This content is hidden until the reader clicks.
:::
`

## Cross-References

Cross-references create automatic numbered labels for figures, tables, sections, and equations.

### Figures

Label a figure with #fig-<label>:

`markdown
![ROC curve comparison](images/roc.png){#fig-roc}

See @fig-roc for the performance comparison.
`

### Tables

Label a table with #tbl-<label>:

`markdown
| Model | Accuracy | F1 |
|-------|----------|----|
| LR    | 0.84     | 0.81 |
| BERT  | 0.91     | 0.90 |

: Model Performance {#tbl-performance}

As shown in @tbl-performance, BERT outperforms logistic regression.
`

### Sections

Label a section with #sec-<label>:

`markdown
## Methodology {#sec-methodology}

...

Refer back to @sec-methodology for details.
`

## Custom SCSS Themes

Quarto supports custom CSS/SCSS for HTML output.

### Creating a Theme File

Create 	heme/custom.scss:

`scss
/*-- scss:defaults --*/
-bg: #fafafa;
-color: #222;
-color: #0066cc;
-family-sans-serif: "Inter", sans-serif;
-font-weight: 600;

/*-- scss:rules --*/
.callout {
  border-radius: 6px;
}

code {
  background-color: #f0f4f8;
  padding: 2px 6px;
  border-radius: 3px;
}
`

### Applying the Theme

`yaml
format:
  html:
    theme: [cosmo, theme/custom.scss]
`

SCSS **defaults** override Bootstrap variables; SCSS **rules** add new CSS.

## Listings Pages

A listings page aggregates multiple documents into a browsable gallery.

### _quarto.yml Listings Entry

`yaml
website:
  navbar:
    left:
      - text: "Labs"
        href: labs/index.qmd

  sidebar:
    - id: labs
      contents: "labs/**"
`

### labs/index.qmd

`markdown
---
title: "Lab Index"
listing:
  contents: "*.qmd"
  type: table
  fields: [title, date, categories, description]
  sort: "date desc"
  filter-ui: true
  sort-ui: true
---
`

## Parameterized Reports

Parameterized reports let you produce multiple outputs from one .qmd file by passing different values at render time.

### Declaring Parameters

`yaml
params:
  ticker: "AAPL"
  start_date: "2024-01-01"
  end_date:   "2024-12-31"
`

### Using Parameters in Code

`python
ticker     = params["ticker"]
start_date = params["start_date"]
`

### Rendering with Different Parameters

`ash
quarto render report.qmd -P ticker:MSFT -P start_date:2023-01-01
`

Or from Python:

`python
import subprocess
tickers = ["AAPL", "MSFT", "GOOGL"]
for t in tickers:
    subprocess.run(["quarto", "render", "report.qmd",
                    f"-P", f"ticker:{t}",
                    "--output", f"report_{t}.html"])
`

## The Freeze Workflow

reeze: auto prevents re-execution of cells when only prose changes.

### How Freeze Works

1. On first render, Quarto executes all cells and stores results in _freeze/.
2. On subsequent renders, if the source code of a cell has not changed, Quarto uses the frozen result.
3. You can commit _freeze/ to version control so collaborators see output without re-running.

### Configuration

`yaml
execute:
  freeze: auto      # freeze unchanged cells
  # freeze: true    # always use frozen output (never re-execute)
`

### Clearing the Cache

`ash
quarto render --cache-refresh   # force re-execution of all cells
`

Or delete the _freeze/ folder for a specific document.

The project-level configuration file controls everything about your Quarto website or book.

### Essential Structure

`yaml

## Publishing to GitHub Pages

### One-Time Setup

`ash
# In your repo root
quarto publish gh-pages
`

This creates (or updates) the gh-pages branch with rendered HTML.

### Automated Publishing with GitHub Actions

Create .github/workflows/publish.yml:

`yaml
name: Publish Quarto Site
on:
  push:
    branches: [main]
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install Quarto
        uses: quarto-dev/quarto-actions/setup@v2
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Publish
        run: quarto publish gh-pages --no-prompt
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
`

### Checking Publication Status

After pushing, navigate to **Settings → Pages** in your GitHub repo to confirm the deployment URL.
